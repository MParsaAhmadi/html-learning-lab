1 HTML Basics ‚Äî Core Building Blocks

Paragraphs

HTML is the standard markup language for creating web pages. It describes the structure of content using a series of elements that tell the browser how to display content.

Each HTML element consists of an opening tag, content, and a closing tag. For example, <p>Hello World</p> creates a paragraph containing "Hello World."

üìù Learning Note: Notice how we used <abbr> for "HTML" above. This element provides the full form of an abbreviation on hover, improving accessibility and understanding.

Accessibility Checklist

‚úÖ Key Accessibility Practices Demonstrated on This Page:
  • Skip navigation link ‚Äî Lets keyboard users bypass repetitive navigation
  • Semantic HTML ‚Äî Screen readers use elements like <nav>, <main>, <article> as landmarks
  • Language declaration ‚Äî lang="en" on the <html> element
  • Heading hierarchy ‚Äî Logical h1‚Üíh2‚Üíh3 structure for navigation
  • Alt text on images ‚Äî Every <img> has descriptive alt
  • Form labels ‚Äî Every input has an associated <label>
  • Color is not the only indicator ‚Äî Information isn't conveyed by color alone
  • Keyboard navigation ‚Äî All interactive elements are keyboard accessible
  • Focus indicators ‚Äî Visible :focus-visible styles on interactive elements
  • ARIA attributes ‚Äî Used where native HTML semantics aren't sufficient
  • Captions/tracks ‚Äî Video and audio include caption tracks
‚ùå Common Accessibility Mistakes:
  • Using <div onclick="..."> instead of <button> (divs aren't keyboard focusable)
  • Missing alt attribute on images (screen readers may read the file name instead)
  • Using placeholder text as the only label (placeholders disappear on focus)
  • Low color contrast (text must have at least 4.5:1 contrast ratio)
  • Removing focus outlines with outline: none without providing an alternative
  • Using ARIA when a native HTML element would work (the first rule of ARIA!)

10 SEO ‚Äî Search Engine Optimization

SEO is the practice of making your website more visible in search engine results. Good HTML structure is the foundation of good SEO.

HTML Elements That Help SEO

SEO-Relevant HTML Elements (Already Used on This Page)
<title>
The most important on-page SEO element. Appears in search results as the clickable headline.
<meta name="description">
Provides the snippet text shown below the title in search results.
<h1> through <h6>
Search engines use headings to understand content hierarchy and topic importance.
<a> with descriptive text
Link text tells search engines what the linked page is about. "Click here" tells them nothing.
<img alt="...">
Alt text makes images indexable and helps them appear in image search results.
<link rel="canonical">
Tells search engines the "official" URL, preventing duplicate content issues.
<time datetime="...">
Machine-readable dates help search engines understand content freshness.
Structured Data (JSON-LD)
Schema.org markup helps search engines create rich snippets (stars, FAQs, breadcrumbs).

‚úÖ SEO Best Practices Summary:
  1. One <h1> per page that matches the <title>
  2. Descriptive, unique <title> for every page (under 60 chars)
  3. Compelling <meta description> (150-160 chars)
  4. Use semantic HTML ‚Äî it's free SEO
  5. Every image has meaningful alt text
  6. Set lang attribute on <html>
  7. Use <link rel="canonical"> to prevent duplicates
  8. Add structured data (JSON-LD) for rich search results
  9. Make link text descriptive (avoid "click here")
  10. Ensure the page is mobile-friendly (viewport meta tag)
